8.1

Introduction to Deployment

Deployment is the process of making a web application available to real users on the internet.

  • Moving code from your local machine to a live server.
  • Ensuring the app runs reliably, handles real traffic, and stays secure.
  • Involves configuration, servers, domains, certificates, and monitoring.

8.2

Deployment Platforms

PaaS (Platform as a Service)

Heroku, Render, Railway, PythonAnywhere. Easy setup, less control.

IaaS (Infrastructure as a Service)

AWS EC2, DigitalOcean, Linode, GCP. Full control of the server.

Serverless

AWS Lambda, Vercel, Cloud Functions. Pay per request, auto-scaling.

Containers

Docker + Kubernetes. Portable, consistent environments.

Choose based on cost, scalability needs, and how much control you want.

8.3

Development vs Production

Development

• Debug mode ON
• Built-in dev server
• Detailed error pages
• Local database / test data
• Hot reload, verbose logs
• Speed of iteration matters

Production

• Debug mode OFF
• Production WSGI server (Gunicorn)
• Generic error pages
• Real database, backups
• Optimized, secure, monitored
• Stability & security matter

⚠ Never run the Flask development server in production.

8.4

Environment Variables & Production Settings

  • Keep secrets out of code (API keys, DB passwords, secret keys).
  • Configure behavior per environment without changing code.
# .env file (never commit this!)
FLASK_ENV=production
SECRET_KEY=super-secret-random-string
DATABASE_URL=postgresql://user:pass@host:5432/db
DEBUG=False
import os
from dotenv import load_dotenv
load_dotenv()

SECRET_KEY = os.environ.get("SECRET_KEY")
DEBUG = os.environ.get("DEBUG", "False") == "True"
8.5

Serving Flask with Gunicorn

Gunicorn (Green Unicorn) is a production WSGI HTTP server that runs multiple worker processes to handle concurrent requests.

# Install
pip install gunicorn

# Run: 4 workers, app object is "app" in app.py
gunicorn --workers 4 --bind 0.0.0.0:8000 app:app

8.6

Setting up HTTPS with Let's Encrypt

HTTPS encrypts traffic between the browser and server. Let's Encrypt provides free, auto-renewable TLS certificates.

# Install Certbot
sudo apt install certbot python3-certbot-nginx

# Obtain & auto-configure certificate for your domain
sudo certbot --nginx -d example.com -d www.example.com

# Test automatic renewal
sudo certbot renew --dry-run
  • Certificates last 90 days and renew automatically.
  • Redirect all HTTP traffic to HTTPS.
8.7

Monitoring & Logging

Logging

Record events, requests, and errors to diagnose issues after they happen.

Monitoring

Track uptime, CPU, memory, response times & alert on problems in real time.

Tools

Sentry, Prometheus + Grafana, Datadog, ELK Stack, UptimeRobot.

import logging
logging.basicConfig(
    filename="app.log", level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s")
app.logger.info("Application started")
8.8

Importance of Security in Deployment

  • Disable debug mode in production — it can leak code & secrets.
  • Enforce HTTPS everywhere; use secure cookies.
  • Store secrets in environment variables, never in the repo.
  • Keep dependencies & the OS patched and updated.
  • Validate input to prevent SQL injection & XSS.
  • Use firewalls, least-privilege access, and regular backups.

⚠ Security is not a one-time step — it is an ongoing responsibility.

SUMMARY

Key Takeaways

  • Deployment makes your app available and reliable for real users.
  • Production differs from development — never ship debug mode.
  • Use environment variables to manage config and secrets.
  • Gunicorn serves Flask apps efficiently in production.
  • Let's Encrypt gives free HTTPS encryption.
  • Monitoring, logging & security are continuous, not optional.
